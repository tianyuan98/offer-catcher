### 修复：简历上传功能不可用

**问题**：`handleResumeUpload` 只接收 `fileName` 字符串，根本没有读取文件内容——上传后用的是硬编码模拟文本，用户真实简历从未被读取

**修复**：
1. `handleResumeUpload(file: File)` 改为接收 `File` 对象，用 `FileReader.readAsText()` 读取文件内容
2. PDF/Word 是二进制格式，`readAsText` 会产生乱码——改为先检测扩展名，PDF/Word 文件 alert 提示用户在 textarea 手动粘贴，仅对 .txt 等纯文本文件执行 FileReader
3. `canAnalyze` 简化为 `resumeText.trim().length > 0 && jdText.trim().length > 10`（不再检查 `resumeName`，因为分析 API 只用 `resumeText`）
4. file input 的 `accept` 添加了 `.txt`，`onChange` 改为传 `file` 对象而非 `file.name`
5. 构建验证通过
